## 5. M-6 — the off-exchange silent-corruption trap (highest-value next lead)

> **ROUND-3 RESOLUTION (2026-05-30): INVESTIGATED + DOCUMENTED-ONLY.** An adversarial
> ground-truth trace CONFIRMED the consumer mis-route at
> `lob-model-trainer/src/lobtrainer/data/dataset.py:472-498` (the SOLE
> `is_regression_strategy` consumer pipeline-wide; gated by `:650-664`). Still
> **DORMANT** — no trainer config consumes a BQP `basic_*` export, and MBO regression
> exports correctly emit `label_strategy="regression"`. `point_return` IS a **ReturnType**
> (LabelFactory dispatch key, `label_factory.py:451`), NOT a LabelingStrategy — so a
> contract-guard on `is_regression_strategy` was **REJECTED** (would leave `get_contract`
> raising on the same string = split-brain SSoT, and masks the BQP producer overload).
> **Pinned** as an executable assertion in
> `test_contract_self_consistency.py::test_point_return_is_not_a_regression_strategy_PINS_CONTRACT`.
> **Fix-of-record: producer-side** — BQP should emit `label_strategy="regression"`
> (`basic-quote-processor/src/export/metadata.rs:344`); execute in a coordinated
> BQP-fusion cycle. The detail below is retained for that cycle.

**Verified facts (hft-contracts):** `is_regression_strategy('point_return')`
returns `False`; `get_contract('point_return')` raises; `point_return`/
`peak_return`/`mean_return` are NOT in `labels._CONTRACTS`.

**The risk (cross-repo, per round-2 Agent 1, NOT yet re-verified in the
consumer):** `basic-quote-processor` emits `label_strategy="point_return"` +
`label_encoding="continuous_bps"` (continuous bps **regression** labels). The
lob-model-trainer `task="auto"` resolver reportedly computes
`task = "regression" if is_regression_strategy(strategy) else "classification"`
→ for `point_return` returns **classification** → would silently train a
**classifier on regression data**. **DORMANT** because every existing BASIC
config sets `task: regression` explicitly.

**Why the fix is non-trivial (design before patching):** `point_return` may be
a **ReturnType** (a sub-axis of regression: point vs smoothed vs peak vs mean)
rather than a **LabelingStrategy** (tlob / triple_barrier / opportunity /
regression). If so, registering it in `_CONTRACTS` (a LabelingStrategy→Contract
map) is the WRONG fix; the right fix is teaching `is_regression_strategy` (and
the trainer resolver) that the BASIC return-types are regression. **Investigate
across 3 repos before touching code:** `hft-contracts/labels.py`,
`basic-quote-processor` (metadata emission), `lob-model-trainer` (task="auto"
resolver in `data/dataset.py`). This is a good FIRST cross-repo task.
